Internship Objective:
The primary objective of the internship was to engage in hands-on cybersecurity learning, apply security tools in controlled scenarios, understand reconnaissance and vulnerability analysis processes, interpret tool outputs, and structure findings into clear, evidence-based documentation. The internship emphasized responsible and ethical security assessment aligned with industry standards and educational development.

Task Breakdown:

Task-1: Web Vulnerability Testing
The first task focused on testing an intentionally vulnerable web application environment using a combination of manual payloads and security tools. SQL Injection behavior was tested manually and validated through automated exploitation frameworks. HTTP headers, server information, and software exposure were also analyzed to understand misconfigurations and weaknesses.

Task-2: Reconnaissance and OSINT Based Enumeration
The second task focused on passive reconnaissance and OSINT-based analysis of a target domain. This included domain ownership inspection, DNS record analysis, SSL certificate investigation, header inspection, redirection behavior analysis, and enumeration of subdomains. This task helped develop an understanding of how reconnaissance informs exploitation and attack surface mapping in real-world scenarios.

Task-3: Final Documentation and Reporting
The third task consolidated the research, findings, screenshots, methodology, and outcomes of Task-1 and Task-2 into a structured internship report. This reinforces the importance of reporting in cybersecurity, where communication and documentation play as significant a role as technical execution.
